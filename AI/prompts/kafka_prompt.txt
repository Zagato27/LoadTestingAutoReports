ЗАДАЧА: Полный аналитический отчет по всем метрикам Kafka  в ходе ступенчатого нагрузочного тестирования

ИСХОДНЫЕ ДАННЫЕ:
- Все данные по kafka_consumer_fetch_rate (для всех тем)
- Все данные по kafka_consumergroup_lag (по всем топикам и группам)
- Все данные по NSI_incident_repository_gauge с репозиторием KafkaRequest
- Любые другие доступные Kafka-метрики

РЕКОМЕНДУЕМЫЕ SLA (по мировой практике):
- Задержка (latency):
  * Нормальная: < 10 мс
  * Предупреждение: 10-100 мс
  * Критическая: > 100 мс
- Пропускная способность (throughput):
  * Должна поддерживаться стабильно при ожидаемой нагрузке
  * Снижение не более чем на 10% при пиковых нагрузках
- Lag консьюмерских групп:
  * Нормальный: < 1000 сообщений
  * Предупреждение: 1000-10000 сообщений
  * Критический: > 10000 сообщений
- Доступность (uptime):
  * Целевая: 99.95% (не более 21.6 минут простоя в месяц)
  * Минимальная: 99.9% (не более 43.2 минут простоя в месяц)

ТРЕБОВАНИЯ К АНАЛИЗУ:
1. Проанализировать ВСЕ доступные метрики полностью, без исключений
2. По каждой метрике предоставить:
   - Полный диапазон значений (мин/макс/средние)
   - Все временные интервалы изменений
   - Значения для всех топиков, групп и клиентов
   - Соответствие рекомендуемым SLA

3. Отдельно выделить:
   - Полный анализ fetch rate по всем потребителям
   - Исчерпывающий анализ lag по всем группам и топикам
   - Детальный анализ всех показателей KafkaRequest

СТРУКТУРА ОТВЕТА:
1. Fetch rate у консумеров
   • Полный анализ по всем client_id без исключения
   • Все зафиксированные значения с временными метками
   • Исчерпывающая информация о динамике изменений
   • Сравнение с целевыми значениями SLA по пропускной способности

2. Lag по потребляющим группам
   • Детальный анализ всех групп и топиков
   • Все значения lag с указанием времени
   • Полная динамика изменений для каждой группы
   • Соответствие целевым SLA по lag

3. Показатель KafkaRequest
   • Исчерпывающий анализ всех данных по gauge
   • Все значения и временные интервалы
   • Полный анализ взаимосвязей с другими метриками
   • Сравнение с ожидаемыми показателями

4. Итоговые выводы
   • Обобщение по всем проанализированным метрикам
   • Выявление всех закономерностей без исключения
   • Конкретные рекомендации на основе всех данных
   • Общая оценка соответствия SLA и предложения по оптимизации

ВАЖНО:
1. НЕ ПРИДУМЫВАТЬ ДАННЫХ. Если в исходных отчетах что-то не указано, отметить это как пробел в данных.
2. НЕ ДЕЛАТЬ ПРЕДПОЛОЖЕНИЙ о причинах и следствиях, если они не очевидны из числовых данных.
3. НЕ ВВОДИТЬ НОВЫХ ТЕРМИНОВ или метрик, которых нет в исходных отчетах.
4. При невозможности сделать вывод из-за недостаточности данных — честно указать на это.
5. Использовать только те числовые значения, которые явно приведены в исходных отчетах.
6. Не пропускайте никакие данные из таблиц. Анализируйте все топики, группы и клиенты, даже если они имеют незначительные показатели.
